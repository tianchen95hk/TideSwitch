# 4Alpha LS Beta — 投资策略报告

---

## 执行摘要

**策略定位**  
- 频率：4 小时一根 K 线（中低频策略）  
- 核心思想：用长期均线划分牛熊，牛市做趋势跟随，熊市做可解释的“恐慌-动量衰减-反转确认”抄底，并可选做空捕捉熊市下跌延续  
- 组合结构：Core-Satellite  
  - Core：BTC（系统性 Beta）  
  - Satellite：按动量选取 Top-N Altcoins（增强收益、分散风险）  
- 可解释性：每一次入场/离场都对应明确的数学条件，并在输出中留下可追溯记录  

**风险与对策**  
- 震荡市假信号：Regime 过滤 + 多指标复合确认 + 止损冷却期  
- 极端波动：硬止损 + 移动止盈 + 波动率折扣（降低仓位）  
- 做空风险：禁止直接翻仓（-1↔1）+ 空头止损/止盈  

---

## 策略假设与适用边界

### 市场假设（可被检验）

- **趋势假设**：价格相对长期均线位置是牛熊 Regime 的有效近似（长期趋势存在性）  
- **恐慌假设**：大回撤区间叠加市场动量扩散显著偏弱时，后续出现更高概率的均值回归/反弹  
- **动量衰减假设**：Y-Index 的回升/走弱用于衡量下跌/上涨动量的“边际变化”，作为反转或延续的确认  

### 不适用场景

- 长期震荡（Regime 反复切换）  
- 高交易摩擦环境（强滑点/高费率/高资金费率）而策略又频繁换仓  
- 小币种极端流动性风险（当前卫星按动量选 Top-N，需结合实际可交易性）

## 核心算法（一页纸概览）

### 1) Regime 判定（长期趋势分界）

- 用 155 天（930 根 4H K）简单移动平均线（MA155）划分牛熊  
- 牛市：价格在 MA155 之上；熊市：价格在 MA155 之下  

### 2) 买入信号（熊市抄底）

熊市 Regime 下，必须同时满足三个条件才触发买入：

- **回撤条件**：价格从近期最高点回撤超过 20%  
- **扩散条件**：扩散指数低于 -20（市场恐慌/广泛走弱）  
- **动量反转条件**：Y-Index 连续 2 根 K 线上行（动量边际转强）  

满足则从空仓（0）进入多头（1）。

### 3) 卖出信号（止盈/风控）

多头持仓期间，优先级从高到低：

- **硬止损**：价格相对入场价下跌 8% 强制平仓，并进入 2 根 K 线冷却期（禁止立即重新开仓）  
- **移动止盈**：记录持仓期间最高价，若从高点回撤超过 10% 则平仓  
- **常规止盈**：布林带上轨突破 + 扩散指数高于 20 + Y-Index 连续 2 根下行  

满足则从多头（1）回到空仓（0）。

### 4) 做空信号（熊市下跌延续）

仅在空仓时、且处于熊市 Regime 才考虑开空：

- **价格结构**：价格在 MA155 之下  
- **扩散条件**：扩散指数高于 20（市场偏单边走弱）  
- **动量延续条件**：Y-Index 连续 2 根 K 线下行（动量边际继续走弱）  

满足则从空仓（0）进入空头（-1）。

### 5) 平空信号

- **价格重新站上 MA155**，或  
- **动量反转**（扩散指数回落 + Y-Index 回升）  

满足则从空头（-1）回到空仓（0）。

### 6) 状态机与决策顺序

每根 4H K 线的决策优先级：

1. 冷却期：若在硬止损冷却区间，强制空仓  
2. 风控：若持仓，先检查止损/止盈  
3. Regime 决策：牛市趋势跟随 / 熊市抄底或做空  
4. 禁止直接翻仓：不允许 -1↔1 同 bar 直接切换，必须先回到 0，下一根 bar 再评估入场  

### 7) 仓位构建（Core-Satellite + 波动率预算）

- **目标波动率**：设定年化目标波动率（TARGET_VOLATILITY）  
- **波动率折扣**：根据当前市场波动率，对总仓位进行缩放（范围 0.5–1.0）  
- **Core 权重**：BTC 权重 = CORE_BTC_WEIGHT × 波动率折扣  
- **Satellite 权重**：Altcoins 总权重 = (1 - CORE_BTC_WEIGHT) × 波动率折扣  
- **卫星选择**：按动量排序取 Top-N，等权分配  
- **做空仓位**：当前实现为 BTC 单品种空头（权重 -1.0）

## 风险控制（止损、止盈、冷却期）

### 1) 硬止损（Long / Short 对称）

- 做多：收益率低于 -8% 触发  
- 做空：收益率低于 -8% 触发  
- 触发后强制平仓，并进入 2 根 K 线冷却期  

### 2) 移动止盈（Trailing Stop）

- 做多：记录持仓期间最高价，若从高点回撤超过 10% 则平仓  
- 做空：记录持仓期间最低价，若从低点反弹超过 10% 则平仓  

### 3) 止损冷却期（Anti-Whipsaw）

硬止损触发后，接下来 2 根 K 线内禁止任何开仓，防止震荡市被反复止损。

## 回测执行与可审计输出

### 1) 终端输出（Terminal）

- 交易事件：`BUY/SELL/SHORT/COVER` 与 `POS` 分两行输出，便于阅读与对齐  
- 调仓事件：`REBAL` 行首对齐输出  
- 进度事件：`EQUITY[...]` 行首对齐输出  

### 2) 输出文件（outputs/）

- `final_result_comparison.png`  
  - 策略净值 vs BTC 基准  
  - BUY/SELL/SHORT/COVER 点位与编号标注  
  - 右侧编号对应的交易操作明细列表  

- `trade_signals.csv`（事件级审计）  
  - 字段：timestamp, action, state, price, reason, from_weights, to_weights, delta_summary, equity  

- `positions_timeseries.csv`（逐 bar 快照）  
  - 字段：timestamp, equity, state, delta_summary, w_SYMBOL...

## 工程实现映射

- **策略主逻辑**：`core_strategy/algo.py :: WhiteboxAlgo.compute_signal`  
  - 输入：当期截面 df_slice + BTC 指标历史 btc_history + 状态/入场价/极值  
  - 输出：目标权重 weights 与下一状态 next_state  

- **回测引擎**：`core_backtest/engine.py :: BacktestEngine.run`  
  - PnL 结算（含资金费率）  
  - 调用策略产生信号  
  - 状态维护（entry/highest/lowest）  
  - 生成可审计输出（trade_signals / positions_timeseries）  

- **可视化**：`run_backtest.py`  
  - 读取回测结果  
  - 绘制净值曲线与编号信号标注  
  - 右侧交易清单展示

## 参数表（Settings）

策略关键参数位于 `config/settings.py`，其中：

- **趋势分界**：`MA_WINDOW = 930`（155 天）  
- **买入阈值**：`BUY_DRAWDOWN = -0.20`, `BUY_DIFFUSION = -20`, `BUY_Y_RISE_DAYS = 2`  
- **卖出阈值**：`SELL_BB_FACTOR = 1.0`, `SELL_DIFFUSION = 20`, `SELL_Y_FALL_DAYS = 2`  
- **风控**：`STOP_LOSS_PCT = 0.08`, `TRAILING_STOP_PCT = 0.10`, `STOPLOSS_COOLDOWN_BARS = 2`  
- **仓位**：`CORE_BTC_WEIGHT = 0.5`, `TOP_N = 3`, `TARGET_VOLATILITY = 0.25`

## 已知限制与改进方向

- **做空仓位过于激进**：当前为 BTCUSDT = -1.0，建议未来引入 SHORT_MAX_WEIGHT 与波动率折扣  
- **交易摩擦建模不足**：可进一步加入滑点模型、费率、资金费率的严格基准，并引入交易频率约束  
- **信号稳定性**：可对 DI/Y 信号引入“连续满足 N 根 bar”或“区间滤波”减少噪声  
- **卫星可交易性**：可加入流动性过滤与黑名单机制，避免小币种异常

## 技术指标简要说明

- **MA155**：155 天简单移动平均线，用于划分牛熊 Regime  
- **回撤**：价格相对近期最高点的跌幅，用于判断恐慌程度  
- **扩散指数（Diffusion Index）**：将 RSI、价格相对均线、均线交叉三类动量信号等权合成，范围 -100 到 100，用于衡量市场广度  
- **Y-Index**：对价格变化率进行滚动标准化后累积，用于衡量趋势强度的边际变化  
- **布林带**：价格围绕均线的波动区间，上轨突破用于判断过热  
- **波动率**：收益率滚动标准差并年化，用于仓位缩放（波动率高时降低仓位）
